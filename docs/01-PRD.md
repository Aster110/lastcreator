# 🧾 产品需求文档（PRD）

## 项目名：神笔 Polyverse（lastcreator）

> ⚠️ **核心约束（2026-04-23 确认）：主要面向手机移动端**
> - 涂鸦画布必须支持 touch 事件（finger drawing）
> - 布局按 375px 竖屏设计，不考虑 PC 适配
> - 按钮/交互区域 ≥ 44px（iOS HIG 最小触控区）
> - 性能目标：低端安卓也能流畅绘画

---

# 一、产品概述

## 1.1 产品定位

一款基于 AI 的即时创造体验产品：
用户通过涂鸦召唤独一无二的宠物，并在末日世界中赋予其生命。

> 核心卖点：
> **“你画的每一笔，都是一条生命”**

---

## 1.2 目标（黑客松版）

### 核心目标

在 10 秒内完成：

```text
涂鸦 → 宠物生成 → 展示 → 情绪冲击
```

### 成功标准（评委视角）

* Demo 稳定（不崩）
* 生成结果有惊喜感
* UI 有“仪式感”
* 体验闭环完整

---

## 1.3 用户画像

| 属性 | 描述           |
| -- | ------------ |
| 年龄 | 16–30        |
| 兴趣 | 创作 / AI / 游戏 |
| 动机 | 表达自我 + 获得反馈  |

---

# 二、核心体验设计

## 2.1 核心用户路径

```text
进入页面
→ 随手涂鸦
→ 点击召唤
→ loading（仪式感）
→ 宠物诞生（高潮）
→ 查看详情
```

---

## 2.2 Wow Moment 设计

关键瞬间：

> “评委随手画一笔 → 10秒生成宠物”

必须满足：

* 有名字
* 有技能
* 有性格
* 有图像

---

# 三、功能设计（模块化）

---

## 3.1 模块一：召唤系统（核心）

### 功能描述

用户绘制涂鸦后，系统生成宠物

---

### 输入

* canvas 图像（base64）

---

### 输出

```json
{
  "id": "uuid",
  "name": "烈焰球兽",
  "habitat": "火域",
  "personality": "暴躁但忠诚",
  "skills": ["火焰冲击", "熔岩护盾", "爆裂突进"],
  "hp": 100,
  "story": "它诞生于世界燃尽后的第一缕火焰",
  "image": "url"
}
```

---

### 子流程

```text
涂鸦
→ Vision解析
→ LLM生成结构
→ 图像生成
→ 聚合返回
```

---

### 规则约束

* 每次生成 ≤ 15 秒
* 输出必须为 JSON
* 失败必须 fallback

---

## 3.2 模块二：宠物展示系统

### 功能描述

将生成结果以视觉方式呈现

---

### 页面结构

#### 宠物卡片

* 图片（主视觉）
* 名字（突出）
* 技能（icon形式）
* 一句话故事

---

### 动效要求

* 卡片展开动画
* 名字高亮出现
* 技能逐条浮现

---

### Loading 文案（必须）

```text
正在解析你的咒文…
生命正在生成…
它正在醒来…
```

---

## 3.3 模块三：图鉴系统（MVP）

### 功能描述

展示用户已创建的宠物

---

### 数据结构

```ts
Pet[]
```

---

### 功能

* 宠物列表（grid）
* 点击查看详情

---

### 存储方案

* localStorage（黑客松推荐）

---

## 3.4 模块四：状态系统（轻量）

### 功能描述

模拟宠物成长

---

### 数据结构

```ts
{
  createdAt: number
  ageDays: number
  stage: "幼年" | "青年" | "成年"
}
```

---

### 规则

| 时间   | 阶段 |
| ---- | -- |
| 1–3天 | 幼年 |
| 4–7天 | 青年 |
| 8天+  | 成年 |

---

👉 MVP只做“展示”，不做复杂逻辑

---

## 3.5 模块五：世界系统（氛围增强）

### 功能描述

增强沉浸感

---

### 内容

* 当前世界天数（动态）
* 总生命值（累计）

---

### 示例

```text
末日第 47 天
世界复苏进度：1287
```

# 四、AI 系统设计（关键）

---

## 4.1 AI Pipeline

```text
涂鸦
↓
Claude Vision（解析）
↓
Claude LLM（生成）
↓
fal.ai / Replicate（图像）
```

---

## 4.2 Prompt 设计

---

### Vision Prompt

```txt
分析这张涂鸦，输出JSON：
{
  "colors": [],
  "shape": "",
  "complexity": ""
}
```

---

### LLM Prompt

```txt
你是末日世界的造物AI。

根据以下信息生成宠物：

{vision_result}

输出JSON：
{
  "name": "",
  "personality": "",
  "habitat": "",
  "skills": ["", "", ""],
  "hp": 100,
  "story": ""
}
```

---

## 4.3 稳定性要求

* 强制 JSON 输出
* 必须校验字段
* 异常 fallback

# 九、风险与解决方案

---

## 风险1：AI延迟或失败

解决：

* fallback宠物
* loading掩盖

---

## 风险2：输出不稳定

解决：

* 强约束 JSON
* prompt固定

---

## 风险3：现场网络问题

解决：

* 预生成数据
* 本地 mock
